# Userflow: UF-005 인라인 기능 중 번역 기능 활용하기

## 1. Flow Boundary

- Begins: 로그인된 사용자가 논문 학습 페이지에서 원문 본문을 읽고 있다.
- Ends: 사용자가 선택한 본문 텍스트의 인라인 번역 결과를 확인한 뒤 결과 창을 닫고 원문 본문 읽기 상태로 돌아온다.
- Assumes: 사용자는 논문 본문 일부를 선택하고 인라인 액션 중 번역을 선택한다. 노트 작성, 인라인 AI 질문, 전체 본문 번역, 번역 실패는 이 flow에 포함하지 않는다.

## 2. Flow Story

로그인된 사용자는 논문 원문을 읽다가 특정 단어나 문장의 의미를 바로 확인하고 싶어 한다. 사용자는 본문 일부를 선택해 나타난 인라인 액션 창에서 번역을 선택하고, 선택한 텍스트 옆에 표시된 번역 결과를 확인한 뒤 다시 원문 읽기 흐름으로 돌아간다.

## 3. Flow Steps

1. **원문 본문 읽기**  
   사용자는 논문 학습 페이지에서 원문 본문을 읽고 있다.

2. **본문 텍스트 선택**  
   사용자가 이해가 필요한 단어나 문장을 드래그해 선택하면 선택 영역 옆에 인라인 액션 창이 나타난다.

3. **인라인 번역 선택**  
   사용자는 인라인 액션 창에서 번역 기능을 선택한다.

4. **번역 결과 확인**  
   선택한 텍스트 옆에 인라인 번역 결과 창이 표시되고 사용자는 원문과 번역 내용을 함께 확인한다.

5. **본문 읽기로 복귀**  
   사용자는 인라인 번역 결과 창을 닫고 원문 본문 읽기 상태로 돌아간다.

## 4. Wireframe References

### Wireframe Flow

```text
WF-008 논문 학습 페이지 - 초기 상태
  -> WF-010 논문 학습 페이지 - 텍스트 선택 액션 상태
  -> WF-011 논문 학습 페이지 - 인라인 번역 결과 상태
  -> WF-008 논문 학습 페이지 - 초기 상태
```

### Referenced Wireframes

| Wireframe | Relevance |
|---|---|
| WF-008 논문 학습 페이지 - 초기 상태 | 사용자가 원문 본문을 읽는 시작 상태이자 인라인 결과를 닫고 돌아오는 종료 상태를 보여준다. |
| WF-010 논문 학습 페이지 - 텍스트 선택 액션 상태 | 본문 일부를 선택했을 때 번역을 포함한 인라인 액션 창이 나타나는 상태를 보여준다. |
| WF-011 논문 학습 페이지 - 인라인 번역 결과 상태 | 선택한 텍스트 옆에 인라인 번역 결과가 표시되는 핵심 결과 상태를 보여준다. |

## 5. Separate Flow Candidates

- 인라인 기능 중 노트 작성하기: 선택한 텍스트를 기반으로 노트를 작성하는 별도 흐름.
- 인라인 기능 중 AI 질문하기: 선택한 텍스트를 맥락으로 AI에게 질문하는 별도 흐름.
- 인라인 번역 결과 닫기 없이 후속 행동하기: 번역 결과를 본 뒤 다른 인라인 액션이나 AI 질문으로 이어지는 흐름.
- 인라인 번역 실패 / 재시도: 선택한 텍스트 번역 요청이 실패했을 때의 복구 흐름.
