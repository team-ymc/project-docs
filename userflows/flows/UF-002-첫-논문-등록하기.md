# Userflow: UF-002 첫 논문 등록하기

## 1. Flow Boundary

- Begins: 로그인된 사용자가 등록된 논문이 없는 빈 서재 페이지에서 논문 등록 진입점을 보고 있다.
- Ends: 새 논문의 업로드와 분석이 완료되어 서재에 완료 상태의 논문 항목이 표시된다.
- Assumes: 사용자는 등록 가능한 PDF 파일을 선택해 업로드한다. 파일 선택, 업로드, 분석 과정에서 오류나 취소가 발생하지 않는다.
- Note: 이 flow는 **happy path만** 다룬다. 분석이 실패했을 때 서재 행에 '실패'를 **표시**하는 것은 MVP 범위이며 FT-002 Story 3가 소유한다(해당 flow는 아직 UF로 없음 — 아래 Separate Flow Candidates 참고). 실패의 재시도·복구는 post-MVP다(FT-003 Out of Scope).

## 2. Flow Story

로그인된 사용자는 비어 있는 서재에서 첫 논문을 추가해 학습을 시작하려고 한다. 사용자는 논문 등록 모달을 열고 PDF 파일을 선택해 업로드를 시작하며, 서재에서 업로드와 분석 진행 상태를 확인한 뒤 완료된 논문 항목을 확인한다.

## 3. Flow Steps

1. **빈 서재에서 시작**  
   로그인된 사용자는 등록된 논문이 없는 서재에서 첫 논문을 등록할 수 있음을 확인한다.

2. **논문 등록 모달 열기**  
   사용자는 논문 등록 버튼을 선택해 PDF 업로드 또는 URL 입력을 선택할 수 있는 등록 모달을 연다.

3. **PDF 업로드 시작**  
   사용자는 PDF 업로드 방식에서 파일을 선택하고 업로드를 시작한다.

4. **업로드와 분석 진행 확인**  
   서재에는 새 논문 항목이 진행 중 상태로 표시되고 사용자는 업로드와 분석이 처리되고 있음을 확인한다.

5. **논문 등록 완료 확인**  
   업로드와 분석이 완료되면 서재의 논문 항목이 완료 상태로 전환되어 첫 논문 등록이 끝난다.

## 4. Wireframe References

### Wireframe Flow

```text
WF-004 서재 페이지 - 빈 상태
  -> WF-005 논문 등록 - 모달
  -> WF-006 서재 페이지 - 논문 업로드 진행 상태
  -> WF-007 서재 페이지 - 논문 업로드 완료 상태
```

### Referenced Wireframes

| Wireframe | Relevance |
|---|---|
| WF-004 서재 페이지 - 빈 상태 | 로그인된 사용자가 첫 논문 등록을 시작하는 빈 서재 상태를 보여준다. |
| WF-005 논문 등록 - 모달 | PDF 업로드 방식과 업로드 실행 상태를 보여준다. |
| WF-006 서재 페이지 - 논문 업로드 진행 상태 | 업로드 후 논문 항목이 진행 중 상태로 서재에 표시되는 중간 상태를 보여준다. |
| WF-007 서재 페이지 - 논문 업로드 완료 상태 | 등록된 논문이 완료 상태로 전환되어 flow가 끝나는 상태를 보여준다. |

## 5. Separate Flow Candidates

- 논문 등록 모달 닫기: 사용자가 논문 등록을 시작하지 않고 모달을 닫아 빈 서재로 돌아가는 흐름.
- URL로 논문 등록하기: 사용자가 PDF 업로드 대신 URL 입력 방식으로 논문 등록을 진행하는 흐름.
- PDF 업로드 실패 / 지원하지 않는 파일: 파일 선택 또는 업로드 과정에서 오류가 발생했을 때의 복구 흐름.
- 논문 분석 실패 / 지연: 업로드 후 분석이 실패하거나 오래 걸릴 때 사용자가 상태를 확인하고 대응하는 흐름.
