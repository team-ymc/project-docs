# Userflow: UF-003 AI 채팅 창에서 논문 질문하기

## 1. Flow Boundary

- Begins: 로그인된 사용자가 논문 학습 페이지에서 논문 본문과 AI 채팅 창을 함께 보고 있다.
- Ends: 사용자가 보낸 질문에 대한 AI 답변이 완료되어 AI 채팅 창에 표시된다.
- Assumes: 사용자는 AI 채팅 창의 질문 입력 영역을 사용한다. 질문 전송과 AI 답변 생성 과정에서 오류, 취소, 응답 중단은 발생하지 않는다.

## 2. Flow Story

로그인된 사용자는 논문을 읽다가 이해가 필요한 내용을 AI 채팅 창에 질문한다. 사용자는 질문을 작성해 전송하고, AI 답변이 대기 상태에서 생성되는 과정을 확인한 뒤 완성된 답변을 읽는다.

## 3. Flow Steps

1. **논문 학습 화면에서 시작**  
   사용자는 논문 본문을 읽으면서 AI 채팅 창의 질문 입력 영역을 확인한다.

2. **질문 작성**  
   사용자는 논문 내용에 대해 궁금한 점을 AI 채팅 창에 입력한다.

3. **질문 전송**  
   사용자가 질문을 전송하면 질문 내용이 대화 영역에 표시되고 AI 답변 대기 상태로 전환된다.

4. **AI 답변 생성 확인**  
   AI 답변이 생성되기 시작하고 사용자는 답변 일부가 나타나는 진행 상태를 확인한다.

5. **완성된 답변 확인**  
   AI 답변 생성이 완료되면 사용자는 AI 채팅 창에서 완성된 답변을 읽을 수 있다.

## 4. Wireframe References

### Wireframe Flow

```text
WF-008 논문 학습 페이지 - 초기 상태
  -> WF-012 논문 학습 페이지 - AI 질문 입력 상태
  -> WF-013 논문 학습 페이지 - AI 응답 대기 상태
  -> WF-014 논문 학습 페이지 - AI 응답 생성 중 상태
  -> WF-015 논문 학습 페이지 - AI 응답 완료 상태
```

### Referenced Wireframes

| Wireframe | Relevance |
|---|---|
| WF-008 논문 학습 페이지 - 초기 상태 | 논문 본문과 AI 채팅 창이 함께 배치된 시작 상태를 보여준다. |
| WF-012 논문 학습 페이지 - AI 질문 입력 상태 | 사용자가 AI 채팅 창에 질문을 작성한 전송 전 상태를 보여준다. |
| WF-013 논문 학습 페이지 - AI 응답 대기 상태 | 질문 전송 후 사용자 질문과 응답 대기 상태가 표시되는 흐름을 보여준다. |
| WF-014 논문 학습 페이지 - AI 응답 생성 중 상태 | AI 답변이 생성 중인 중간 상태를 보여준다. |
| WF-015 논문 학습 페이지 - AI 응답 완료 상태 | 완성된 AI 답변이 AI 채팅 창에 표시된 종료 상태를 보여준다. |

## 5. Separate Flow Candidates

- AI 답변 생성 중단하기: 사용자가 답변 생성 중 일시정지 또는 중단 행동을 선택하는 흐름.
- AI 답변 실패 / 재시도: 질문 전송 후 AI 답변 생성에 실패했을 때 사용자가 재시도하거나 복구하는 흐름.
- 후속 질문 이어가기: 완성된 답변 이후 같은 AI 채팅 창에서 추가 질문을 이어가는 흐름.
- 인라인 기능에서 AI 질문하기: 본문 선택 후 인라인 액션을 통해 AI 질문을 시작하는 별도 흐름.
