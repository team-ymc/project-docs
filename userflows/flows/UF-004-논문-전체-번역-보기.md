# Userflow: UF-004 논문 전체 번역 보기

## 1. Flow Boundary

- Begins: 로그인된 사용자가 논문 학습 페이지에서 원문 본문을 보고 있으며, 전체 번역 버튼을 사용할 수 있다.
- Ends: 사용자가 한글 번역 본문을 확인한 뒤 원문 보기로 돌아와 논문 학습 페이지의 원문 상태를 다시 보고 있다.
- Assumes: 사용자는 현재 논문 본문 전체 번역 기능을 켰다가 다시 원문 보기로 전환한다. 번역 로딩, 실패, 언어 선택 변경, 인라인 번역은 이 flow에 포함하지 않는다.

## 2. Flow Story

로그인된 사용자는 논문 원문을 읽다가 전체 내용을 한국어로 확인하고 싶어 한다. 사용자는 논문 학습 페이지에서 전체 번역 기능을 선택해 한글 번역 본문을 확인한 뒤, 다시 원문 보기로 돌아와 기존 학습 맥락을 이어간다.

## 3. Flow Steps

1. **원문 학습 화면에서 시작**  
   사용자는 논문 학습 페이지에서 현재 섹션의 원문 본문과 전체 번역 버튼을 확인한다.

2. **전체 번역 선택**  
   사용자는 번역 버튼을 선택해 현재 논문 본문을 한글 번역 상태로 전환한다.

3. **한글 번역 본문 확인**  
   논문 본문 영역이 한글 번역문으로 바뀌고, 사용자는 동일한 학습 화면 안에서 번역된 내용을 읽는다.

4. **원문 보기로 복귀**  
   사용자는 번역 상태를 다시 전환해 논문 본문 영역을 원문 상태로 되돌린다.

## 4. Wireframe References

### Wireframe Flow

```text
WF-008 논문 학습 페이지 - 초기 상태
  -> WF-009 논문 학습 페이지 - 한글 번역 상태
  -> WF-008 논문 학습 페이지 - 초기 상태
```

### Referenced Wireframes

| Wireframe | Relevance |
|---|---|
| WF-008 논문 학습 페이지 - 초기 상태 | 사용자가 원문 본문을 읽으며 전체 번역 기능을 선택할 수 있는 시작 상태이자, 번역 확인 후 원문으로 돌아온 종료 상태를 보여준다. |
| WF-009 논문 학습 페이지 - 한글 번역 상태 | 원문 영역이 한글 번역 본문으로 전환되어 사용자가 번역 내용을 확인하는 중간 상태를 보여준다. |

## 5. Separate Flow Candidates

- 번역 로딩 / 실패 처리: 전체 번역 요청 후 로딩이 길어지거나 실패했을 때의 상태 확인 및 복구 흐름.
- 번역 언어 선택하기: 사용자가 한국어 외 다른 번역 언어를 선택할 수 있는 경우의 별도 흐름.
- 인라인 번역 활용하기: 전체 본문이 아니라 선택한 문장이나 단어만 번역하는 별도 흐름.
