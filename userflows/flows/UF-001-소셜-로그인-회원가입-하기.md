# Userflow: UF-001 소셜 로그인 / 회원가입 하기

## 1. Flow Boundary

- Begins: 비로그인 사용자가 Paper Teacher 랜딩 페이지에서 로그인 / 회원가입 진입점을 보고 있다.
- Ends: 소셜 인증이 완료되어 사용자가 로그인된 상태로 빈 서재 페이지에 도착한다.
- Assumes: 사용자는 Google, Kakao, 또는 Naver 중 하나로 인증을 완료한다. 인증 중 오류, 계정 선택 취소, 약관 관련 차단은 발생하지 않는다.

## 2. Flow Story

비로그인 사용자는 Paper Teacher 랜딩 페이지에서 서비스를 시작하기 위해 로그인 / 회원가입을 선택한다. 사용자는 열린 인증 모달에서 원하는 소셜 계정을 선택해 인증을 완료하고, 로그인된 상태로 자신의 서재에 진입한다.

## 3. Flow Steps

1. **랜딩 페이지에서 시작**  
   비로그인 사용자는 Paper Teacher의 핵심 소개와 로그인 / 회원가입 버튼을 확인한다.

2. **인증 모달 진입**  
   사용자는 로그인 / 회원가입 버튼을 선택해 소셜 인증 방법을 고를 수 있는 모달을 연다.

3. **소셜 계정 선택**  
   사용자는 Google, Kakao, 또는 Naver 중 하나를 선택해 인증을 진행한다.

4. **서재 진입 완료**  
   인증이 완료되면 모달이 닫히고 로그인된 사용자는 빈 서재 페이지에 도착한다.

## 4. Wireframe References

### Wireframe Flow

```text
WF-001 랜딩 페이지 - 비로그인
  -> WF-003 회원가입 / 로그인 모달
  -> WF-004 서재 페이지 - 빈 상태
```

### Referenced Wireframes

| Wireframe | Relevance |
|---|---|
| WF-001 랜딩 페이지 - 비로그인 | 비로그인 사용자가 인증 흐름을 시작하는 랜딩 상태를 보여준다. |
| WF-003 회원가입 / 로그인 모달 | 사용자가 소셜 인증 방법을 선택하는 핵심 인증 상태를 보여준다. |
| WF-004 서재 페이지 - 빈 상태 | 인증 완료 후 사용자가 도착하는 로그인 상태의 첫 서재 화면을 보여준다. |

## 5. Separate Flow Candidates

- 인증 모달 닫기: 사용자가 인증을 진행하지 않고 모달을 닫아 비로그인 랜딩 페이지로 돌아가는 흐름.
- 소셜 인증 실패 / 취소: 소셜 인증 과정에서 실패하거나 사용자가 외부 인증을 취소했을 때의 복구 흐름.
- 기존 로그인 사용자의 서재 진입: 이미 로그인된 사용자가 랜딩 페이지에서 내 서재로 이동하는 흐름.
