# Feature: FT-002 서재

## 1. Overview

- Summary: 로그인 사용자의 등록된 논문 목록을 서재 화면에 보여주고, 빈 상태·목록·등록 진입점·학습 진입을 제공한다.
- Status: Draft
- Tracking: -
- Depends on: FT-001 Auth

## 2. Feature Boundary

### TASK In Scope

- 서재 빈 상태 (등록 진입점 노출)
- 사용자별 논문 목록 조회 / 렌더 (WF-016)
- 논문 행 상태 표시 (진행 중 / 완료)
- 논문 선택 → 학습 페이지(FT-004) 진입

#### DRAFT
  - 인증 사용자의 논문만 조회 (FT-001 세션 기반)
  - 빈 서재(WF-004): 등록 버튼/안내 노출
  - 목록(WF-016): 논문 행 렌더, 행 상태(processing/ready)는 FT-003이 만든 데이터를 표시
  - 행 선택 → WF-008 학습 페이지로 라우팅
  - 진행 중 행 상태 갱신 수신 방식은 FT-003의 상태 전달(polling/SSE)과 연동

### TASK Out of Scope

- 논문 등록 / 분석 (FT-003)
- 논문 검색 / 정렬 / 필터, 폴더 · 태그 분류
- 논문 삭제 / 이름 변경 / 메타 편집

#### DRAFT
  - 등록·분석 파이프라인은 FT-003 소유 (서재는 결과 행만 렌더)
  - 검색/정렬/필터, 폴더링, 다중 선택 관리
  - 항목 삭제/리네임 등 목록 편집

## 4. Stories

### Story 1. 빈 서재에서 등록 진입 `MVP`

- Story: 로그인 사용자로서, 등록된 논문이 없을 때 빈 서재에서 첫 논문 등록을 시작하고 싶다.
- type: USER
- Source Userflows: UF-002 Step 1-2
- Acceptance Criteria:
  - 등록된 논문이 없는 사용자는 빈 서재(WF-004)와 등록 진입점을 본다.
  - 등록 진입점 선택 시 FT-003의 등록 모달로 전환된다.
- Depends on: -

### Story 2. 등록된 논문 목록 보기 `MVP`

- Story: 로그인 사용자로서, 내가 등록한 논문들을 서재 목록에서 보고 싶다.
- type: USER
- Source Userflows: UF-002 Step 5
- Acceptance Criteria:
  - 세션의 사용자 소유 논문만 목록(WF-016)에 표시된다(타 사용자 논문 비노출).
  - 각 논문 행이 상태(진행 중/완료)와 함께 표시된다.
- Depends on: FT-001 Story 2

### Story 3. 논문 등록·분석 진행 상태 보기 `MVP`

- Story: 로그인 사용자로서, 등록한 논문의 분석 진행과 완료 상태를 서재에서 보고 싶다.
- type: USER
- Source Userflows: UF-002 Step 4-5
- Acceptance Criteria:
  - 업로드 직후 논문이 진행 중 상태(WF-006)로 서재에 표시된다.
  - 분석이 완료되면 행이 완료 상태(WF-007)로 갱신된다.
- Depends on: Story 2

### Story 4. 논문 선택해 학습 진입 `MVP`

- Story: 로그인 사용자로서, 서재에서 논문을 선택해 학습 페이지로 들어가고 싶다.
- type: USER
- Source Userflows: UF-002 Step 5
- Acceptance Criteria:
  - 완료(ready) 상태 논문 행을 선택하면 해당 논문의 학습 페이지 초기 상태(WF-008)로 이동한다.
- Depends on: FT-004 Story 1

## 5. Done Criteria

- 빈 상태 → 등록 진입 → 목록 표시 → 학습 진입까지 서재 흐름이 동작한다.
- 사용자별 논문만 노출되고, 행 상태가 분석 진행에 따라 갱신된다.
