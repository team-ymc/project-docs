# Feature: FT-006 인라인 설명 · 번역

## 1. Overview

- Summary: 본문 텍스트를 선택하면 인라인 액션을 띄우고, 선택 영역에 대한 번역·문맥 기반 설명을 같은 자리에 보여준다.
- Status: Draft
- Tracking: -
- Depends on: FT-004 논문 학습 뷰어

## 2. Feature Boundary

### In Scope

- 본문 텍스트 드래그 선택 → 인라인 액션 창 (WF-010)
- 인라인 번역 결과 (WF-011)
- 인라인 설명 (선택 영역의 문맥 기반 설명)
- 결과 창 닫기 → 본문 읽기 복귀

### Out of Scope

- 인라인 노트 작성 → 후속(post-MVP) (UF-005 Separate Flow 후보)
- 인라인 AI 질문 → FT-007 (인라인은 트리거만)
- 인라인 번역/설명 실패·재시도 → 후속(post-MVP)

## 3. Userflow References

| Userflow | Relation |
|---|---|
| UF-005 인라인 기능 중 번역 기능 활용하기 | 이 Feature가 직접 구현하는 flow |

## 4. Stories

### Story 1. 사용자는 본문 일부를 선택해 인라인 액션을 띄울 수 있다 `MVP`

- type: USER
- Source Userflows: UF-005 Step 1-2
- Acceptance Criteria:
  - 본문 텍스트를 드래그하면 선택 영역 옆에 인라인 액션 창(WF-010)이 나타난다.
- Depends on: FT-004 Story 2

### Story 2. 사용자는 선택한 텍스트의 번역을 같은 자리에서 보고 닫을 수 있다 `MVP`

- type: USER
- Source Userflows: UF-005 Step 3-5
- Acceptance Criteria:
  - 액션 창에서 번역 선택 시 선택 영역 옆에 인라인 번역 결과(WF-011)가 표시된다.
  - 결과 창을 닫으면 원문 읽기 상태(WF-008)로 돌아간다.
- Depends on: Story 1

### Story 3. 시스템은 선택 텍스트와 문맥의 번역을 생성해 제공한다 `MVP`

- type: SYSTEM
- Source Userflows: UF-005 Step 3-4
- Acceptance Criteria:
  - 선택 텍스트(+주변 문맥)를 입력으로 번역 결과를 생성해 인라인으로 제공한다.
- Depends on: FT-003 Story 3

> **Deferred (UF 미작성)** — 인라인 설명: 선택 텍스트의 문맥 기반 설명. UF 작성 후 Story로 추가. (capability·In Scope는 유지)

## 5. Done Criteria

- 선택 → 액션 → 인라인 번역/설명 → 닫기까지 동작한다.

## 6. Open Questions

- 설명의 컨텍스트 범위: 선택 문장만 vs 주변 문단 vs 위키(FT-008) 연계
- 인라인 액션 메뉴 구성 항목 (번역/설명/AI 질문/노트)과 선택 단위(단어/문장/문단)
- 선택 텍스트 번역·설명도 외부 AI 서버 호출 — FT-005와 동일한 BE↔AI 서버 seam(contracts 공유).
