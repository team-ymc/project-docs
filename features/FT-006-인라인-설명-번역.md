## 4. Stories

### Story 1. 텍스트 선택 → 인라인 액션 `MVP`

- Story: 로그인 사용자로서, 본문 일부를 선택해 바로 쓸 수 있는 인라인 액션을 띄우고 싶다.
- type: USER
- Source Userflows: UF-005 Step 1-2
- Acceptance Criteria:
  - 본문 텍스트를 드래그하면 선택 영역 옆에 인라인 액션 창(WF-010)이 나타난다.
- Depends on: FT-004 Story 2

### Story 2. 인라인 번역 `MVP`

- Story: 로그인 사용자로서, 선택한 텍스트의 번역을 같은 자리에서 보고 닫고 싶다.
- type: USER
- Source Userflows: UF-005 Step 3-5
- Acceptance Criteria:
  - 액션 창에서 번역 선택 시 선택 영역 옆에 인라인 번역 결과(WF-011)가 표시된다.
  - 결과 창을 닫으면 원문 읽기 상태(WF-008)로 돌아간다.
- Depends on: Story 1

### Story 3. BE → AI 서버: 선택 텍스트 번역 요청 `MVP`

- Story: 시스템으로서, 선택 텍스트와 그 문맥의 번역을 AI 서버에 요청해 받는다.
- type: SYSTEM
- Source Userflows: UF-005 Step 3-4
- Acceptance Criteria:
  - BE가 선택 텍스트(+주변 문맥)를 입력으로 AI 서버에 번역을 요청하고 결과를 수신한다. (BE → AI 서버)
- Depends on: FT-003 Story 3

> **Deferred (UF 미작성)** — 인라인 설명: 선택 텍스트의 문맥 기반 설명. UF 작성 후 Story로 추가. (capability·In Scope는 유지)

## 5. Done Criteria

- 선택 → 액션 → 인라인 번역/설명 → 닫기까지 동작한다.
