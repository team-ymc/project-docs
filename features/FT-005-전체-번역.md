## 4. Stories

### Story 1. 전체 번역 토글 `MVP`

- Story: 로그인 사용자로서, 논문 본문 전체를 한글 번역으로 보고 다시 원문으로 돌아가고 싶다.
- type: USER
- Source Userflows: UF-004 Step 1-4
- Acceptance Criteria:
  - 번역 버튼 선택 시 본문 영역이 한글 번역 상태(WF-009)로 전환된다.
  - 번역 버튼을 다시 선택하면 원문 상태(WF-008)로 복귀한다.
  - 번역 본문은 원문의 섹션·문단 구조를 유지한다.
- Depends on: FT-004 Story 2

### Story 2. BE → AI 서버: 본문 번역 요청 `MVP`

- Story: 시스템으로서, 논문 본문의 한글 번역을 AI 서버에 요청해 받아 학습 화면에 제공한다.
- type: SYSTEM
- Source Userflows: UF-004 Step 2-3
- Acceptance Criteria:
  - BE가 구조화 본문을 입력으로 AI 서버에 번역을 요청하고 결과를 수신한다. (BE → AI 서버)
  - 동일 논문 재요청 시 일관된 번역을 제공한다(캐싱 정책은 Open Questions).
- Depends on: FT-003 Story 3

## 5. Done Criteria

- 원문 ↔ 한글 토글이 구조를 유지한 채 동작한다.

## 6. Open Questions

- 캐싱 정책과 번역 단위(문단 단위 점진 표시/스트리밍 여부)
