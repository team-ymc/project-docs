# Feature: FT-005 전체 번역

## 1. Overview

- Summary: 학습 페이지에서 논문 본문 전체를 한글 번역 상태로 토글하고 다시 원문으로 복귀한다.
- Status: Draft
- Tracking: -
- Depends on: FT-004 논문 학습 뷰어

## 2. Feature Boundary

### In Scope

- 전체 번역 토글 (원문 ↔ 한글, WF-008 ↔ WF-009)
- 번역 본문 렌더 (구조 단위 유지)

### Out of Scope

- 번역 언어 선택 (한국어 외) → 후속(post-MVP) (UF-004 Separate Flow 후보)
- 번역 로딩 / 실패 처리 → 후속(post-MVP)
- 인라인(부분) 번역 → FT-006

## 3. Userflow References

| Userflow | Relation |
|---|---|
| UF-004 논문 전체 번역 보기 | 이 Feature가 직접 구현하는 flow |

## 4. Stories

### Story 1. 사용자는 논문 본문 전체를 한글 번역으로 보고 다시 원문으로 돌아갈 수 있다 `MVP`

- type: USER
- Source Userflows: UF-004 Step 1-4
- Acceptance Criteria:
  - 번역 버튼 선택 시 본문 영역이 한글 번역 상태(WF-009)로 전환된다.
  - 번역 버튼을 다시 선택하면 원문 상태(WF-008)로 복귀한다.
  - 번역 본문은 원문의 섹션·문단 구조를 유지한다.
- Depends on: FT-004 Story 2

### Story 2. 시스템은 논문 본문을 한글로 번역해 학습 화면에 제공한다 `MVP`

- type: SYSTEM
- Source Userflows: UF-004 Step 2-3
- Acceptance Criteria:
  - 구조화 본문을 입력으로 한글 번역 결과를 생성해 학습 화면에 제공한다.
  - 동일 논문 재요청 시 일관된 번역을 제공한다(캐싱 정책은 Open Questions).
- Depends on: FT-003 Story 4

## 5. Done Criteria

- 원문 ↔ 한글 토글이 구조를 유지한 채 동작한다.

## 6. Open Questions

- 번역 생성 시점: 분석 단계에서 미리 생성 vs 요청 시 on-demand
- 캐싱 정책과 번역 단위(문단 단위 점진 표시/스트리밍 여부)
- 번역 생성은 외부 AI 서버 호출 — 생성 시점 결정 후 BE↔AI 서버 seam을 contracts로 박는다.
